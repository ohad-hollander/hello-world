# Plan 04-02: Game Flow — Wave Progression, Game-Over Screen, and Restart

## Objective

Complete the Space Invaders game loop by implementing: (1) wave progression — destroying all aliens starts a new wave with slightly faster march speed (FLOW-01); (2) a proper game-over screen showing the final score with a restart prompt (FLOW-02); (3) clean restart via `initGame()` with no timer leaks — all timers and state reset fully (FLOW-03). Also adds wave number to the HUD.

## Must-Haves

- Destroying all 55 aliens triggers a 2-second "WAVE CLEAR!" pause, then starts Wave 2 with aliens reset and march speed increased by ~5%
- Each new wave increments the wave counter; wave number is shown in the HUD
- Shields are NOT reset between waves (authentic Space Invaders behavior — damage persists)
- "GAME OVER" screen shows final score and "Press ENTER or R to restart" — displayed when all lives are lost
- Pressing ENTER or R calls `initGame()` — returns to Wave 1 with score=0, lives=3, clean formation
- After restart, no leftover speed, stale timers, or visual artifacts from the previous game

## Context

**Current `game.js` state (after Plan 04-01):**
- `gameState` values: `'playing'` | `'game_over'` | `'wave_clear'`
- `update(dt)` only processes `updatePlaying(dt)` — `game_over` and `wave_clear` are frozen stubs
- `renderGameOver()` exists but shows placeholder text `'(full restart in Phase 4)'`
- `renderWaveClear()` exists but shows placeholder text `'(wave progression in Phase 4)'`
- `initGame()` was extended in Plan 04-01 to include `initUFO()` and `playerBullet = null`

**`updatePlaying()` already sets `gameState = 'wave_clear'` when `getAliveCount() === 0`.** Plan 04-02 fills in what happens next.

**Existing globals in game.js:** `score`, `lives`, `gameState`. This plan adds `waveNumber` and `waveClearTimer`.

**`initAliens()` in aliens.js** resets the full formation to 55 aliens at `FORMATION_TOP=120`. It also resets `march.dx`, `march.stepTimer`, `march.pendingDown`, `march.stepInterval` to initial values.

**Wave speed bonus:** After `initAliens()` resets `march.stepInterval` to the base value for 55 aliens, we apply a wave multiplier to make each successive wave slightly faster. `calcStepInterval(55) = 0.05 + (55/55) * 0.95 = 1.0`. Per-wave multiplier: `Math.pow(0.95, waveNumber - 1)` (5% faster per wave, Wave 1 = 1.0×, Wave 2 = 0.95×, Wave 3 = 0.9025×, etc.).

**`keys` state map** is defined in input.js and available globally. Restart listens for `keys['Enter']` or `keys['KeyR']`.

**`ctx.textAlign` persistence:** Always set `ctx.textAlign` explicitly before each `fillText()` call to avoid inheriting previous state.

**Part A — Add new update functions after the existing `updatePlaying()` function in game.js:**

After the closing `}` of `updatePlaying(dt)`, append:

```javascript
function updateWaveClear(dt) {
  // FLOW-01: count down, then start next wave
  waveClearTimer += dt;
  if (waveClearTimer >= WAVE_CLEAR_DELAY) {
    startNextWave();
  }
}

function startNextWave() {
  waveNumber++;
  waveClearTimer = 0;

  // Reset aliens to full 55-alien formation
  initAliens();

  // Apply wave speed bonus: each wave, march step interval is WAVE_SPEED_FACTOR faster
  // initAliens() resets march.stepInterval to base (1.0s for 55 aliens)
  // Override it with the wave multiplier (clamp to minimum 0.05s)
  march.stepInterval = Math.max(0.05, calcStepInterval(55) * Math.pow(WAVE_SPEED_FACTOR, waveNumber - 1));

  // Reset UFO timer for the new wave
  initUFO();

  // Clear alien bullets — no stale projectiles carried over
  alienBullets.length = 0;

  // NOTE: Shields are intentionally NOT reset — authentic behavior: shield damage persists across waves
  // NOTE: Player position and lives are intentionally NOT reset between waves

  gameState = 'playing';
}

function updateGameOver(dt) {
  // FLOW-02/03: listen for restart key — Enter or R
  if (keys['Enter'] || keys['KeyR']) {
    initGame(); // full clean restart — all timers and state reset
  }
}
```
